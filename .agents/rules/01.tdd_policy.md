# TDD (Test-Driven Development) 원칙 및 작업 흐름

본 프로젝트는 엄격한 TDD(테스트 주도 개발) 방법론을 따릅니다. AI 에이전트는 기능 개발 시 다음 규칙을 **반드시** 준수해야 합니다.

## 1. TDD 작업 사이클 (Red-Green-Refactor)
- **선(先) 테스트 작성:** 실제 비즈니스 로직(`src/` 폴더)을 구현하거나 수정하기 전에, **반드시 `test/` 폴더에 유닛 테스트 코드를 먼저 작성**하십시오.
- **테스트 자동 실행 및 검증:** 테스트 코드를 작성한 후, 에이전트가 직접 `uv run pytest` 명령어를 사용하여 테스트를 실행하고 의도대로 실패(Red)하는지 먼저 확인하십시오.
- **기능 구현 및 통과:** 테스트가 실패하는 것을 확인한 후 본 로직을 구현하고, 다시 테스트를 실행하여 통과(Green)하는 것을 확인하십시오.
- **코드 스타일 및 린팅 (Refactor):** 모든 테스트를 통과한 후, `uv run ruff check --fix .` 및 `uv run ruff format .` 명령어를 실행하여 코드 스타일을 정리하고 린트 오류를 수정한 뒤 작업을 마무리하십시오.

## 2. 테스트 프레임워크 및 구조
- 모든 테스트 클래스는 반드시 파이썬 표준 라이브러리인 `unittest.TestCase`를 상속받아 작성하십시오.

## 3. 기능 세분화 및 테스트 단위 분리 (가장 중요)
구현할 기능을 한 번에 거대한 테스트 함수로 뭉뚱그려 작성하지 마십시오. **반드시 하나의 비즈니스 요구사항을 가장 작은 논리 단위로 세분화하여, 각각 독립된 테스트 함수(단계별 TDD)로 작성**해야 합니다.

**[💡 세분화 작성 예시 (Pattern to follow)]**
* **요구사항:** "주식을 매수한 후 15일 이상 지났는데 수익률(변화율)이 +3% ~ -3% 이내라면, 상승 동력이 없는 종목으로 판단하여 전량 매도한다."
* **잘못된 예:** 위 조건을 하나의 `test_sell_condition` 함수에서 모두 테스트함.
* **올바른 예 (이 프로젝트의 규칙):** 아래와 같이 조건을 쪼개어 개별 테스트로 작성함.
  - `test_is_over_15_days_since_buy()`: 매수일과 오늘 날짜를 비교하여 15일 경과 여부만 판단하는 테스트
  - `test_is_within_sideways_yield_range()`: 주가 변화율이 +3% ~ -3% 구간(횡보) 이내인지 수치 계산만 판단하는 테스트
  - `test_should_sell_sideways_stock()`: 위의 두 조건이 모두 True일 때 '전량 매도' 시그널을 정확히 반환하는지 종합 판단하는 테스트
