# Ruff 정책

본 프로젝트는 일관된 코드 스타일과 높은 품질을 유지하기 위해 **Ruff**를 사용합니다. AI 에이전트와 매수/매도 로직 개발 시 다음 규칙을 반드시 준수해야 합니다.

## 1. 코드 스타일 및 린팅 (Ruff)
- **표준 포맷터:** Ruff의 `format` 기능을 사용하며, 기본 설정은 `Black` 스타일을 따릅니다.
  - 문자열 따옴표: Double quotes (`"`) 사용
  - 들여쓰기: Space 4개 사용
  - 행 끝: 자동(`auto`) 감지
- **린팅 규칙:** `pyproject.toml`에 정의된 `select` 규칙을 모두 준수해야 합니다.
  - `E`, `F`, `B`, `I`, `UP` 등의 주요 규칙 활성화
  - `isort` 기능이 포함되어 있으므로 임포트 순서는 자동으로 정렬됩니다.

## 2. 주요 명령어
- **이미 작성된 코드 전체 점검 및 수정:**
  ```bash
  uv run ruff check --fix .
  uv run ruff format .
  ```

## 3. 예외 사항
- **테스트 파일 (`test/**`):** `assert` 구문 사용이 허용됩니다 (`S101` 무시).
- **`__init__.py`:** 사용되지 않는 임포트(`F401`)가 허용됩니다.
- **라인 길이:** 최대 88자를 기본으로 하되, 포맷터가 자동으로 처리하는 범위 내에서 유연하게 관리합니다.
