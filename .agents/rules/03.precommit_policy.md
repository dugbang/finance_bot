# Pre-commit 정책

본 프로젝트는 일관된 코드 품질을 유지하기 위해 **Pre-commit**을 사용합니다. AI 에이전트는 커밋 전 다음을 확인해야 합니다.

## 1. 자동화 및 검증 (Pre-commit)
- **커밋 전 자동 검사:** 모든 커밋 시 `pre-commit` 훅이 실행됩니다.
  - `trailing-whitespace`: 행 끝 불필요한 공백 제거
  - `end-of-file-fixer`: 파일 끝 개행 추가
  - `ruff-check`: 린트 오류 체크 및 가능한 항목 자동 수정 (`--fix`)
  - `ruff-format`: 전체 코드 포맷팅 자동 적용
- **에이전트 의무:** 코드 수정 후에는 항상 직접 린트와 포맷팅을 실행하여 커밋 가능한 상태인지 확인하십시오.

## 2. 주요 명령어
- **수동으로 모든 파일에 pre-commit 실행:**
  ```bash
  uv run pre-commit run --all-files
  ```
